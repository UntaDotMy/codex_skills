# Real-World Benchmarking and Familiarity

## Purpose

Improve UX decisions using proven real-world patterns while preserving product fit.

## Benchmarking Protocol

1. Select 2–4 relevant references:
   - Similar domain product
   - Mature design system pattern source
   - Accessibility-focused public service or standards-aligned product
2. Extract:
   - Navigation model
   - Task flow structure
   - Error handling and recovery patterns
   - Onboarding and progressive disclosure patterns
3. Convert findings into principles and guardrails, not cloned screens.
4. Tie each principle to a concrete user story or journey stage.

## Familiarity and Consistency Rules

- Reuse familiar interaction patterns for common tasks.
- Keep labels, actions, and behavior consistent across user journeys.
- Avoid novelty-driven interaction changes without evidence.
- Prefer incremental UX changes with measurable impact.
- Use progressive disclosure for advanced functionality to protect novice usability.

## Risk Checks

Flag as risk when:

- A single-page optimization breaks journey continuity.
- New patterns conflict with established mental models.
- Visual/interaction decisions are driven by trends without user-value evidence.
- Benchmark references are listed without showing transferable principles.
